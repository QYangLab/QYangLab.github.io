Dokka-style 4D heading / object-motion stimulus demo

This standalone HTML now mirrors the full 4D surrogate used in shuyuchen/: x_t^W = [v_x^self, v_z^self, v_x^obj, v_z^obj] and o_t = [o_x^bg, o_z^bg, o_x^obj, o_z^obj] with o^bg = -v^self and o^obj = v^obj - v^self. The canvas is split into a world-frame velocity plane, an observer-frame cue plane, and separate x/z history panels so both components stay visible at once.

Controls

Heading-related world-frame component v_x^self. Positive values correspond to rightward heading.
Forward world-frame component v_z^self. This drives the common negative background cue o_z^bg.
World-frame lateral object velocity v_x^obj. Positive values move the object rightward in the world.
World-frame depth object velocity v_z^obj. Positive values move the object forward in the world.
Number of cue dots used in the observer-frame panel and trace density in the 4D display.

World-frame causes and observer-frame cues

World x/z
The upper-left panel shows the world-frame velocity plane. The blue-cyan vector is self-motion; the orange-red vector is object motion.
Observer x/z
The upper-right panel shows egocentric cues: the background arrow is -v^self and the object arrow is v^obj - v^self.
Channel history
The lower panels keep all four scalar channels visible. Solid lines are world velocities; dashed lines are observer-frame cues.
Ambiguity
If the object is stationary in the world, then o^obj = o^bg = -v^self: the object can still move in the observer frame without moving in the world.
